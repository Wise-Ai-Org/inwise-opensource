## Codebase Patterns
- NeDB (`@seald-io/nedb`) does NOT support `$setOnInsert` — use `findOneAsync` then `insertAsync`/`updateAsync` for idempotent upsert semantics.
- Collections live under `app.getPath('userData')`: `meetings.db`, `tasks.db`, `people.db`, `voiceprints.db`. Calendar events are persisted into the `meetings` collection with `source: 'calendar'` and `calendarEventId` as the external key.
- Build/quality checks: `npx tsc -p tsconfig.main.json --noEmit` for main, `npx tsc -p tsconfig.renderer.json --noEmit` for renderer, `npm run build` for full build. No test runner is configured in package.json.
- Main-process module layout: `src/main/` (main.ts wires IPC, database.ts owns NeDB, calendar-watcher.ts owns ICS polling/in-memory cache, preload.ts exposes `inwiseAPI`).
- Renderer layout: `src/renderer/` (App.tsx root, Sidebar.tsx records, Settings.tsx manages voiceprints, Badge.tsx captures audio).
- Audio playback from Chromium requires the user-gesture chain: `new Audio(url)` and `.play()` must happen synchronously in the click handler with no `await` between them. Prefetch IPC blobs on mount and store `URL.createObjectURL` results in a `useRef<Map>` cache; always `.catch()` the promise returned by `.play()` and surface the rejection.
- Preferred error color: `var(--red)` (used across Onboarding.tsx, Settings.tsx). Teal accent: `var(--teal)`.

- **Learnings for future iterations:**
  - Chromium's autoplay policy rejects `audio.play()` if there's ANY `await` between the click event and the call. The original code `await` on IPC then `new Audio().play()` always broke the gesture chain — prefetch is the correct fix.
  - `audio.play()` returns a Promise in modern browsers; it MUST have a `.catch()` or the rejection is silently swallowed (this is the exact bug we hit).
  - `onerror` handler uses `audio.error?.code` (MediaError numeric code 1-4), not a string name.
  - When restoring refs after callbacks fire, always guard with `if (audioRef.current === audio)` — a later click may have already swapped the ref.
  - Per-id transient UI state (inline errors, missing-audio flags) works well as `Record<string, T>` + `Record<string, number>` timer map; don't add an effect when a ref-keyed `setTimeout` will do.
  - `var(--red)` is the project-wide error color; `var(--teal)` is the accent.

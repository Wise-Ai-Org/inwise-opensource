## Codebase Patterns
- Tests live next to source as `X.test.ts` with a `run()` export and `if (require.main === module) run()`. Build via `npx tsc -p tsconfig.main.json`, then run `node dist/main/X.test.js`. No `npm test` script exists.
- `src/main/config.ts` uses electron-store. `electron-store` cannot be imported from compiled JS at arbitrary locations — it must run from within the project's own `dist/main/` so node_modules resolves. Factor pure logic out of store-dependent helpers for unit tests.
- Main-process typecheck: `npx tsc -p tsconfig.main.json --noEmit` (silent = success). Renderer: `npx tsc -p tsconfig.renderer.json --noEmit`.
- `preload.ts` hardcodes an `allowed[]` channel allowlist — every new main→renderer push channel must be added or it's silently dropped.
- Date→epoch-ms is the convention for main→renderer IPC payloads carrying Date objects.
- `mainWindow` hides on close (preventDefault+hide); fallback UI-path checks must verify `mainWindow.isVisible()`, not just `!isDestroyed()`.
- `import { app } from 'electron'` in plain node is safe at load-time (`app` is `undefined`); it only fails when you actually call an `app.*` method. This means tests can import modules that import electron without booting electron.
- NeDB (`@seald-io/nedb`) without a `filename` option is pure in-memory — perfect for unit tests. `{ field: null }` matches documents where the field is null OR missing (Mongo-compatible), so additive schema changes need no migration.
- For testable DB helpers on `database.ts`, use the `__setTasksDbForTests(db)` injection hook. Follow the same pattern if future stories need to exercise meetings/people DBs in unit tests.

Learnings for future iterations:
- Pure-function extraction is the key testability pattern in this codebase. `staleness-sweep.ts` does this via `sweepStaleTasks(now?: Date)` (injects the clock); `welcome-back.ts` takes it further and injects ALL dependencies. When a new story needs data from multiple modules (config + db + calendar + staleness), follow this pattern and let `main.ts` be the wiring layer.
- The PRD null-return condition "`welcomeBackLastSeenAt >= lastOpenedAt`" MUST compare against the module-load snapshot from config.ts, NOT the live store value. `markAppOpened()` fires on every `show` event, so the live `lastOpenedAt` is always >= `welcomeBackLastSeenAt` and the gate would never fire. `getLastOpenedAtSnapshot()` is the correct reader. Document this pattern wherever else "pre-session state" is needed.
- Contradictions are stored in `meetings.insights.contradictions` by `saveInsights` — do NOT re-call `detectContradictions` (that's the LLM extractor, it's a COMPUTE function not a STORE). Query `getMeetings()` and filter by `insights.contradictions.length > 0` + createdAt window.
- `getLoginItemSettings` can throw on non-darwin/win32 platforms; wrap in try/catch and default to `false`. US-009 will add the `setLoginItemSettings` side. The PRD's guard ("never return launchAtStartupOffer if already openAtLogin") is in compute, not in the IPC — good for testing, but means the compute also needs a try-safe boolean input.
- `calendarWatcher.getUpcomingEvents()` returns events including the PAST 7 days (see `fetchIcsEvents` starting `todayStart` 7 days back). That's why missed-meetings detection can iterate upcomingEvents rather than needing a separate history query. Document this in AGENTS.md for calendar-watcher if a future story assumes future-only.
- The new preload entries (`welcomeBackCompute`, `welcomeBackDismiss`) are both `invoke` style and do NOT need to be added to the `allowed[]` array in the `on()` handler — that allowlist is ONLY for main→renderer push channels. This distinction is documented in Codebase Patterns as well.
- When adding a new module with `any[]` dependencies (tasks, meetings), default to `any` at the boundary but use well-named interfaces for the OUTPUT shape (WelcomeBackResult, WelcomeBackWins, WelcomeBackAsk). The IPC payload is the contract for US-005's UI.
- Wins fields are individually optional (`?`) — only populated when count > 0. Empty wins + no ask is the signal for US-005's "Nothing urgent while you were out" empty-state copy.

## Codebase Patterns
- Tests live next to source as `X.test.ts` with a `run()` export and `if (require.main === module) run()`. Build via `npx tsc -p tsconfig.main.json`, then run `node dist/main/X.test.js`. No `npm test` script exists.
- `src/main/config.ts` uses electron-store. `electron-store` cannot be imported from compiled JS at arbitrary locations ‚Äî it must run from within the project's own `dist/main/` so node_modules resolves. Factor pure logic out of store-dependent helpers for unit tests.
- Main-process typecheck: `npx tsc -p tsconfig.main.json --noEmit` (silent = success).
- `preload.ts` hardcodes an `allowed[]` channel allowlist ‚Äî every new main‚Üírenderer push channel must be added or it's silently dropped.
- Date‚Üíepoch-ms is the convention for main‚Üírenderer IPC payloads carrying Date objects.
- `mainWindow` hides on close (preventDefault+hide); fallback UI-path checks must verify `mainWindow.isVisible()`, not just `!isDestroyed()`.
